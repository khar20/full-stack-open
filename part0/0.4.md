```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: The user types a new note and presses the "Save" button.

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note
    Note right of browser: The browser sends the form data to the server.
    
    activate server
    server-->>browser: (HTTP 302 Found) Redirect to /exampleapp/notes
    Note left of server: The server asks the browser to redirect to the notes page.
    deactivate server

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/notes
    Note right of browser: The browser follows the redirect and requests the HTML for the notes page.
    
    activate server
    server-->>browser: (HTTP 200 OK) HTML document for 'notes'
    Note left of server: The server sends the base HTML file.
    deactivate server

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.css
    Note right of browser: The browser requests the CSS file.

    activate server
    server-->>browser: (HTTP 200 OK) 'main.css' file
    Note left of server: The server sends the stylesheet.
    deactivate server

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/main.js
    Note right of browser: The browser requests the JavaScript file.
    
    activate server
    server-->>browser: (HTTP 200 OK) 'main.js' file
    Note left of server: The server sends the JavaScript code.
    deactivate server

    Note right of browser: The browser executes the JavaScript, which requests the note data from the server.

    browser->>server: GET https://studies.cs.helsinki.fi/exampleapp/data.json
    Note right of browser: The browser fetches the notes data.

    activate server
    server-->>browser: (HTTP 200 OK) 'data.json' file
    Note left of server: The server sends the updated list of notes.
    deactivate server

    Note right of browser: The browser renders the list of notes.
```
